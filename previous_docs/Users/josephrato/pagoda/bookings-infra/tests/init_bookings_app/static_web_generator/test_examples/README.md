# Service Highlights Visual Regression Test Examples

This directory contains example HTML files generated by `test_template.py`. These files serve as visual regression test baselines and can be opened in a browser for manual verification.

## How It Works

When you run `test_template.py`, the test:

1. **Loads complete preview input data** from `complete_preview_input.json` (all preview input fields)
2. **Loads complete database data** from `complete_database_data.json` (all database return values)
3. **Creates a complete PreviewInputModel** from the JSON preview input data
4. **Merges it with database data** using `merge_preview_with_sql_data()`
5. **Renders the complete HTML template** using `TemplateRenderer.render_website_template()`
6. **Saves the HTML file** to this `test_examples/` directory

This produces the same HTML output that the `generate_preview` endpoint would create when given complete input data.

## Data Files

The test uses two JSON files that contain complete data structures:

- **`complete_preview_input.json`**: Contains all preview input fields needed to populate every optional section of the template. This matches the structure expected by `PreviewInputModel.from_dict()`.

- **`complete_database_data.json`**: Contains all database return values that would be returned by `StaticWebGeneratorRepository.get_company_data()`. This includes company info, services, staff, contact details, location data, etc.

These JSON files are based on the reference data in `app/docs/COMPLETE_TEMPLATE_DATA_REFERENCE.md` and ensure that all template variables and sections are populated with actual data (no placeholders).

## Purpose

These example files demonstrate what the complete website looks like when rendered with all values provided, specifically showcasing:

- Complete website structure and layout
- Service highlights section with Tabler icons
- All optional sections (testimonials, gallery, socials, etc.)
- Styling and responsive behavior
- HTML rendering with `|safe` filters

## Usage

### Viewing Examples

Simply open any HTML file in this directory in your web browser:

```bash
# Example:
open test_examples/generate_preview_complete_example.html
```

### Viewing HTML Files in Browser

You can view HTML files in your browser using the simple HTTP server:

```bash
# From the test_examples directory
cd tests/init_bookings_app/static_web_generator/test_examples

# View generate_preview example (default)
python html_editor_server.py

# View populate_site_template example
python html_editor_server.py --type populate_site_template

# Specify a custom port
python html_editor_server.py --port 8000 --type generate_preview
```

Then open your browser to `http://localhost:8000/`

**Available types:**

- `generate_preview` - Shows `generate_preview_complete_example.html` (default)
- `populate_site_template` - Shows `populate_site_template_complete_example.html`

### Visual Regression Testing

1. **Initial Generation**: Run the test to generate baseline example

   ```bash
   # From project root
   python -m pytest tests/init_bookings_app/static_web_generator/test_template.py -v -s
   ```

2. **After Changes**: Re-run the test to generate new example
   - Compare new HTML file with previous version (using diff or visual comparison)
   - Verify visual changes are intentional
   - Commit updated file if changes are expected

3. **Manual QA**: Open file in browser to manually verify:
   - Complete website renders correctly
   - Service highlights icons display properly
   - Layout and styling are correct
   - Responsive behavior works
   - All sections are present

## File Structure

The tests generate:

- `generate_preview_complete_example.html` - Complete website HTML generated via `generate_preview` endpoint flow (uses `PreviewInputModel`)
- `populate_site_template_complete_example.html` - Complete website HTML generated via `populate_site_template` endpoint flow (uses `TemplateInput`)

## Generated File Format

The HTML file is a **complete, standalone website** that includes:

- Full HTML structure (`<!DOCTYPE html>`, `<head>`, `<body>`)
- All embedded CSS styles
- All embedded JavaScript
- All sections (hero, services, service highlights, about us, testimonials, gallery, etc.)
- Service highlights with Tabler icons properly rendered

This means you can open the file directly in a browser without needing additional assets or a server.

## Notes

- **Files are auto-generated**: Don't manually edit these files - they're regenerated on each test run
- **Version control**: Commit these files to track visual changes over time
- **Intentional changes**: When you make intentional visual changes, re-run the test and commit the updated file
- **Complete example**: This file represents what `generate_preview` produces with all optional fields provided

## Running the Test

**Important:** Run these commands from the project root directory (`/Users/josephrato/pagoda/bookings-infra`).

### Default Test Example

```bash

python -m pytest tests/init_bookings_app/static_web_generator/test_template.py::TestGeneratePreviewTemplate::test_generate_preview_with_all_values_saves_html_example -v -s
```

Or using `pytest` directly (if installed):

```bash
pytest tests/init_bookings_app/static_web_generator/test_template.py::TestGeneratePreviewTemplate::test_generate_preview_with_all_values_saves_html_example -v -s
```

The test will print a message showing where the HTML file was saved:

```
✓ Saved HTML example to: /path/to/test_examples/generate_preview_complete_example.html
```
